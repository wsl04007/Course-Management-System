# Campus-Run-System（校园跑步打卡系统）
## 项目概述
本项目旨在开发一款适配校园场景的跑步打卡系统，解决校园跑步管理、运动数据统计、校园体育活动组织等问题，目标用户为全体在校师生及校园体育管理工作人员。系统聚焦校园体育管理的核心痛点，实现跑步打卡、数据统计、活动组织等功能的线上化、智能化，提升校园体育管理效率，助力学生养成良好的运动习惯。

## 核心功能模块
### 1. 跑步打卡管理
支持学生在线打卡，管理员可设置打卡时间范围、校园打卡区域，系统自动校验打卡有效性（杜绝校外打卡、代打卡等作弊行为）；学生可查看个人打卡记录，包括打卡时间、打卡地点、跑步里程等信息，未打卡时可收到提醒。

### 2. 运动数据统计
系统自动记录学生每次跑步的里程、时长、配速、消耗热量等核心数据，生成个人运动报表，支持按周、月、学期查看数据趋势；管理员可查看全体学生或班级的运动数据统计，掌握学生运动情况，为体育教学提供参考。

### 3. 体育活动组织
管理员可发布校园跑步相关活动（如校园跑挑战赛、月度跑步打卡活动等），设置活动时间、参与规则、奖励机制；学生可在线报名参与活动，系统自动统计活动参与人数、完成情况，活动结束后生成活动总结报表，方便管理员复盘。

### 4. 用户信息管理
区分学生、教师、管理员三类角色，实现权限分级管理：
- 学生：可修改个人信息、重置密码、查看个人打卡及运动数据；
- 教师：可查看所带班级学生的运动数据、打卡情况，协助管理员管理学生；
- 管理员：可批量导入/导出用户信息、分配用户角色、管理用户账号状态，便捷开展用户管理工作。

### 5. 系统设置
管理员可灵活配置系统核心参数，包括打卡规则（每日打卡次数、单次跑步最低里程）、数据统计标准、消息通知方式（打卡提醒、活动通知）等，适配不同校园的体育管理需求；支持系统日志查看，便于排查操作问题。
